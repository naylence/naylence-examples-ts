Naylence Biomes Example — Browser Client

A biome is a logical enclave in the fabric: a sentinel plus the internal agents and (optionally) child biomes it shields behind a single, secure boundary.

This example demonstrates hierarchical fabric topology with multiple biomes and comprehensive routing:

  • Main sentinel with two child sentinel biomes (tree topology)
  • Each child biome hosts a math agent
  • Deep-to-deep delegation: Agent in one biome calls agent in another
  • Client only connects to main sentinel, addresses route automatically
  • In real deployments, each biome is also a security enclave; this example intentionally disables security so you can focus purely on topology and routing.

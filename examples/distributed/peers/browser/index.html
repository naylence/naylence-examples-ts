Naylence Peers Example — Browser Client

This example demonstrates a peer-to-peer topology where two sentinels connect as peers (no parent/child hierarchy). Unlike the Biomes example, there's no containment here: each sentinel is a peer that can discover and route to the other's agents.

The client connects to Peer Sentinel 1; the fabric routes calls transparently over the peer link to whichever sentinel hosts the target agent.

  • Peer Sentinel 1 routes calls to the agent 1 at math1@fame.fabric (add, multiply operations)
  • Peer Sentinel 2 routes calls to the agent 2 at math2@fame.fabric (streaming Fibonacci)
  • Client connects only to Peer Sentinel 1; routing across the peer link is fully transparent
  • Address propagation happens automatically between peers (no centralized registry)
  • Envelope inspector shows outbound and inbound messages for this client
